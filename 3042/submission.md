## STEP 1 อธิบายโจทย์

**โจทย์**
รับเลขจำนวนเต็มบวกมา 1 ตัว เรียกว่า N แล้วไล่ดูตัวเลขตั้งแต่ N ลงมาจนถึง 0 ว่ามีตัวไหนบ้างที่หาร 10 ลงตัว เจอตัวไหนก็พิมพ์ออกมา คั่นด้วยช่องว่าง

**Input**
1 บรรทัด เป็นจำนวนเต็มบวก

**Output**
1 บรรทัด เป็นเลขที่หาร 10 ลงตัว เรียงจากมากไปน้อย คั่นด้วยช่องว่าง

**Constraints**
- Timeout 1 วินาที
- Memory 32000 bytes

## STEP 2 สิ่งที่สังเกต

ข้อนี้ตรงไปตรงมาที่สุดในบรรดาที่ทำมา แต่มีจุดเล็ก ๆ ที่ถ้าไม่อ่านดี ๆ จะพลาด

จุดแรกคือทิศทาง โจทย์เขียนว่า "ตั้งแต่ N ถึง 0" ไม่ใช่ "0 ถึง N" แปลว่าต้องนับถอยหลัง ผลลัพธ์เรียงจากมากไปน้อย ถ้าเผลอวนขึ้นจะได้ลำดับกลับด้าน ซึ่งน่าจะไม่ผ่าน

จุดที่สองคือเลข 0 มันหาร 10 ลงตัวด้วยนะ เพราะ 0 หาร 10 เหลือเศษ 0 ตรงนี้ผมต้องคิดอยู่แป๊บนึงว่าจะนับหรือไม่นับ แต่โจทย์บอกให้ไล่ถึง 0 เลย ผมเลยนับรวมไปด้วย

จุดที่สามคือถ้า N น้อยกว่า 10 เช่น N เป็น 7 มันจะไม่มีเลขไหนหาร 10 ลงตัวเลยนอกจาก 0 ผลลัพธ์ก็จะมีแค่ตัวเดียว

edge case ที่นึกออก
- N เป็น 0
- N น้อยกว่า 10
- N เป็นเลขที่หาร 10 ลงตัวพอดี ต้องนับตัวมันเองด้วย
- N ไม่ลงตัว ต้องเริ่มจากตัวถัดลงมาที่ลงตัว

## STEP 4 วิธีที่ใช้จริง

ตามแผนเดิม ไม่ได้เปลี่ยนอะไร แค่ตอนเขียนจริงมีสองจุดที่ต้องนั่งคิด

จุดแรกคือ range ตอนนับถอยหลัง ผมเขียน `range(a, -1, -1)` ซึ่งอ่านครั้งแรกงงมาก เพราะมีเลข -1 อยู่สองที่ ที่จริงมันคนละความหมายกัน ตัวกลางคือจุดที่จะหยุด ตัวหลังคือก้าวเดินทีละเท่าไหร่ ผมต้องใส่ -1 ตรงตัวกลางเพราะ range มันไม่เอาตัวสุดท้าย ถ้าใส่ 0 มันจะหยุดที่ 1 แล้วไม่พิมพ์ 0 ออกมา ตอนแรกผมเขียนเป็น `range(a, 0, -1)` จริง ๆ แล้วลองรันดูถึงเห็นว่า 0 หายไป

จุดที่สองคือการพิมพ์ ปกติ print มันขึ้นบรรทัดใหม่ให้ทุกครั้ง แต่โจทย์ต้องการบรรทัดเดียว ผมเลยใส่ `end=" "` เข้าไปเพื่อบอกว่าให้จบด้วยช่องว่างแทน

ส่วนเงื่อนไข ผมเขียน `if not i % 10` ซึ่งความหมายคือถ้าเศษเป็น 0 เพราะ 0 มันถูกมองว่าเป็นเท็จ พอเอา not มาครอบก็กลายเป็นจริง เขียนแบบนี้มันสั้นดี แต่พออ่านย้อนกลับมาผมว่า `if i % 10 == 0` อ่านเข้าใจง่ายกว่า ทำงานเหมือนกันเป๊ะ ถ้าเขียนใหม่ผมคงเลือกอันหลัง

```python
a = int(input())

for i in range(a, -1, -1):
    if not i % 10:
        print(i, end=" ")
```

มีอีกเรื่องที่ผมสังเกตเห็นแต่ไม่ได้แก้ คือผลลัพธ์มันจะมีช่องว่างค้างอยู่ท้ายสุดเสมอ เพราะทุกตัวที่พิมพ์จะต่อท้ายด้วยช่องว่างหมด รวมถึงตัวสุดท้ายด้วย ผมส่งไปแล้วผ่าน เลยเดาว่าระบบตรวจมันตัดช่องว่างท้ายบรรทัดทิ้งให้ แต่ผมยังไม่แน่ใจตรงนี้ว่าทุกระบบจะเป็นแบบนั้นไหม

- Expected: `10 0`
- Actual: `10 0`
- Result: ผ่าน

**ชุดที่ 3 — N น้อยกว่า 10**

- Expected: `0`
- Actual: `0`
- Result: ผ่าน
