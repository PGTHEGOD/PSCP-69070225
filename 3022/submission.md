## STEP 1 อธิบายโจทย์

**โจทย์**
รับค่าอุณหภูมิมา 1 ค่า พร้อมบอกว่าตอนนี้เป็นหน่วยอะไร และอยากได้เป็นหน่วยอะไร แล้วแปลงให้ หน่วยที่มีคือ C, F, K, R

**Input**
3 บรรทัด
- บรรทัด 1 ค่าอุณหภูมิ เป็นจำนวนจริง
- บรรทัด 2 หน่วยต้นทาง เป็นตัวอักษรตัวเดียว
- บรรทัด 3 หน่วยปลายทาง เป็นตัวอักษรตัวเดียว

**Output**
1 บรรทัด เป็นอุณหภูมิในหน่วยปลายทาง ทศนิยม 2 ตำแหน่ง

**Constraints**
- Timeout 1 วินาที
- Memory 32000 bytes

## STEP 2 สิ่งที่สังเกต

อย่างแรกที่ผมนับดูคือมันมีคู่การแปลงกี่แบบ หน่วยต้นทาง 4 แบบ ปลายทาง 4 แบบ ก็คือ 16 คู่ ถ้าเขียน if ไล่ทีละคู่คงตายเพราะต้องจำสูตร 12 สูตร (อีก 4 คู่คือหน่วยเดิมไปหน่วยเดิม)

แต่ตารางที่โจทย์ให้มาเป็นสูตรแปลงจาก C ไปหน่วยอื่นทั้งหมด ผมเลยคิดว่าถ้าเอา C เป็นตัวกลางน่าจะง่ายกว่ามาก คือแปลงอะไรก็ตามเข้ามาเป็น C ก่อน แล้วค่อยแปลงจาก C ออกไปหน่วยที่ต้องการ แบบนี้เหลือแค่ 8 สูตร และแต่ละสูตรเขียนแยกกันไม่ปนกัน

พอลองคิดต่อก็เห็นว่าวิธีนี้มันเผื่ออนาคตด้วย ถ้าโจทย์เพิ่มหน่วยที่ 5 มา ก็เขียนเพิ่มแค่ 2 สูตร ไม่ใช่เพิ่มทีละ 8 คู่

จุดที่ต้องระวัง
- ค่าที่รับมาเป็นจำนวนจริง ไม่ใช่จำนวนเต็ม ต้องใช้ float ไม่ใช่ int
- ต้องพิมพ์ทศนิยม 2 ตำแหน่งเสมอ แม้คำตอบจะลงตัว
- กรณีหน่วยต้นทางกับปลายทางเป็นตัวเดียวกัน ต้องคืนค่าเดิมออกไปเฉย ๆ
- สูตรของ R ผมต้องเช็กดี ๆ เพราะมันไม่ได้อิงกับ C ตรง ๆ แต่อิงกับ K

ตรงเรื่องการอ่าน input ผมยังไม่แน่ใจตรงนี้ ว่าไฟล์ทดสอบจะมีช่องว่างท้ายบรรทัดติดมาหรือเปล่า ถ้ามีแล้วผมไม่ได้ตัดทิ้ง การเทียบตัวอักษรมันจะไม่ตรงเลย แล้วผลลัพธ์จะออกมาเป็น 0.00 ทั้งที่โค้ดไม่ได้ error อะไร ผมส่งไปแล้วผ่าน เลยเดาว่าไม่มี แต่ถ้าให้ปลอดภัยจริง ๆ ควรใส่ strip ไว้

## STEP 3 แผนแรก

Step 1 รับอุณหภูมิ แปลงเป็นเลขทศนิยม

Step 2 รับหน่วยต้นทาง แล้วรับหน่วยปลายทาง

Step 5 ดูว่าหน่วยปลายทางคืออะไร แล้วแปลงจากเซลเซียสไปเป็นหน่วยนั้น

Step 6 พิมพ์ผลลัพธ์ให้เหลือทศนิยม 2 ตำแหน่ง

## STEP 4 วิธีที่ใช้จริง

ทำตามแผนเป๊ะ ไม่ได้เปลี่ยนเลย เพราะพอเขียนออกมาแล้วมันแบ่งเป็นสองก้อนสวย ๆ ก้อนบนคือขาเข้า ก้อนล่างคือขาออก อ่านแล้วเข้าใจง่ายกว่าที่คิดไว้อีก

สูตรที่ใช้
- ขาเข้า F มา C ใช้ ลบ 32 แล้วคูณ 5 หาร 9
- ขาเข้า K มา C ใช้ ลบ 273.15
- ขาเข้า R มา C ใช้ ลบ 491.67 แล้วคูณ 5 หาร 9
- ขาออก C ไป F ใช้ คูณ 9 หาร 5 แล้วบวก 32
- ขาออก C ไป K ใช้ บวก 273.15
- ขาออก C ไป R ใช้ บวก 273.15 ก่อน แล้วค่อยคูณ 9 หาร 5

ตัวที่ผมงงที่สุดคือ R ตอนแรกผมเกือบเขียนเป็น `c * 9 / 5 + 491.67` เพราะเห็นว่ามันคล้าย F แต่พอลองแทนค่า 0 องศา C ดู มันควรได้ 491.67 ถ้าใช้สูตรผิดจะได้ 491.67 เหมือนกันพอดี เลยไม่รู้ว่าผิด ผมเลยลองแทน 100 องศา C ดูอีกค่า สูตรถูกต้องได้ 671.67 ส่วนสูตรที่ผมจะเขียนผิดได้ 671.67 เหมือนกัน

พอลองแล้วได้เท่ากันสองรอบผมเลยไปนั่งกระจายสมการดู ถึงรู้ว่ามันเท่ากันจริง ๆ เพราะ 273.15 คูณ 9 หาร 5 ได้ 491.67 พอดี เลยเขียนแบบไหนก็ถูกทั้งคู่ แต่ผมเลือกเขียนแบบบวก 273.15 ก่อนแล้วคูณ เพราะมันสื่อความหมายตรงกว่า คือ R มันคือ K ที่ถูกยืดสเกลออกไป

ตัวแปร `c` กับ `result` ผมตั้งค่าเริ่มต้นเป็น 0 ไว้ก่อน เผื่อกรณีที่ตัวอักษรที่รับมาไม่ตรงกับเงื่อนไขไหนเลย โปรแกรมจะได้ไม่พังเพราะตัวแปรไม่มีค่า

```python
temp = float(input())
from_unit = input()
to_unit = input()
c = 0
result = 0
if from_unit == "C":
    c = temp
elif from_unit == "F":
    c = (temp - 32) * 5 / 9
elif from_unit == "K":
    c = temp - 273.15
elif from_unit == "R":
    c = (temp - 491.67) * 5 / 9

if to_unit == "C":
    result = c
elif to_unit == "F":
    result = c * 9 / 5 + 32
elif to_unit == "K":
    result = c + 273.15
elif to_unit == "R":
    result = (c + 273.15) * 9 / 5

print(f"{result:.2f}")
```
