## STEP 1 อธิบายโจทย์

**โจทย์**
ร้านอาหารคิดเงินลูกค้าแบบนี้ เอาค่าอาหารมาคิดค่าบริการ 10% ก่อน แต่ค่าบริการนี้มีพื้นกับเพดาน คือถ้าคิดออกมาได้น้อยกว่า 50 บาท ให้เก็บ 50 บาท ถ้าเกิน 1000 บาท ให้เก็บแค่ 1000 บาท จากนั้นเอาค่าอาหารบวกค่าบริการมารวมกัน แล้วค่อยคิด VAT 7% ทับอีกที ตอบว่าลูกค้าต้องจ่ายเท่าไหร่

**Input**
1 บรรทัด เป็นจำนวนเต็ม คือค่าอาหารและเครื่องดื่ม

**Output**
1 บรรทัด เป็นยอดที่ต้องจ่าย ทศนิยม 2 ตำแหน่ง

**Constraints**
- Input เป็นจำนวนเต็ม
- Timeout 1 วินาที
- Memory 32000 bytes

## STEP 2 สิ่งที่สังเกต

โจทย์นี้ไม่ยากที่การคำนวณ แต่ยากที่ลำดับ ต้องอ่านให้ดีว่า VAT คิดจากอะไร โจทย์เขียนไว้ชัดว่าคิดจากราคาที่รวมค่าบริการแล้ว ไม่ใช่คิดจากค่าอาหารอย่างเดียวแล้วเอามาบวกทีหลัง ถ้าทำสลับกันตัวเลขจะเพี้ยน

เรื่องที่สองคือพื้นกับเพดานของค่าบริการ ผมลองคิดย้อนดูว่ามันจะเริ่มมีผลตอนไหน
- ถ้าค่าอาหารต่ำกว่า 500 บาท 10% จะไม่ถึง 50 ก็เลยโดนดันขึ้นเป็น 50
- ถ้าค่าอาหารเกิน 10000 บาท 10% จะเกิน 1000 ก็เลยโดนตัดเหลือ 1000
- ระหว่าง 500 ถึง 10000 คือช่วงที่คิด 10% ตรง ๆ ได้เลย

พอรู้ตัวเลขสองตัวนี้ ตอนทำ test case ก็ง่ายขึ้นเยอะ เพราะรู้ว่าควรไปทดสอบตรงไหน

อีกจุดคือ output ต้องเป็นทศนิยม 2 ตำแหน่งเสมอ ต่อให้คำตอบลงตัวก็ต้องมี .00 ต่อท้าย ไม่ใช่พิมพ์เป็นจำนวนเต็มเปล่า ๆ

## STEP 4 วิธีที่ใช้จริง

ทำตามแผนเดิมทั้งหมด ไม่ได้เปลี่ยนอะไรเลย เพราะโจทย์มันเป็นสูตรตรง ๆ อยู่แล้ว ไล่ทีละบรรทัดตามที่โจทย์เล่ามาก็ได้คำตอบ

มีสองอย่างที่ผมตัดสินใจตอนเขียนจริง

อย่างแรก การเช็กพื้นกับเพดาน ผมใช้ if กับ elif แทนที่จะเขียน if สองอันแยกกัน เพราะค่าบริการมันเป็นได้แค่อย่างเดียว จะน้อยกว่า 50 และมากกว่า 1000 พร้อมกันไม่ได้อยู่แล้ว ใช้ elif เลยพอ

อย่างที่สอง ตอนบวก VAT ผมไม่ได้เขียนแยกว่าคิด vat = subtotal * 0.07 แล้วค่อยเอามาบวก แต่คูณ 1.07 ทีเดียวจบ ผลลัพธ์เหมือนกันแต่บรรทัดน้อยกว่า

ส่วนการพิมพ์ ผมใช้ f-string แบบ `:.2f` เพราะมันจัดทศนิยมให้ครบ 2 ตำแหน่งเอง ไม่ต้องไปนั่ง round แล้วมาแปลงเป็น string เองอีกรอบ ถ้าใช้ round เฉย ๆ แล้วคำตอบเป็น 1177.0 มันจะพิมพ์ออกมาแค่หลักเดียว ซึ่งน่าจะไม่ผ่าน

```python
amount = int(input())

service_charge = amount * 0.10
if service_charge < 50:
    service_charge = 50
elif service_charge > 1000:
    service_charge = 1000

subtotal = amount + service_charge
total = subtotal * 1.07

print(f"{total:.2f}")
```

## STEP 5 Test Case

**ชุดที่ 1 — ช่วงกลาง คิด 10% ตรง ๆ**
เอาเลขกลม ๆ มาลองก่อน ค่าบริการไม่ชนพื้นไม่ชนเพดาน
- Input: `1000`
- Expected: `1177.00`
- Actual: `1177.00`
- Result: ผ่าน
- คิดมือ: ค่าบริการ 100 รวมเป็น 1100 บวก VAT ได้ 1177

**ชุดที่ 2 — ยอดน้อย โดนดันขึ้นเป็น 50**
10% ของ 200 คือ 20 ซึ่งไม่ถึงขั้นต่ำ ถ้าใครลืมเงื่อนไขนี้จะได้ 235.40 แทน
- Input: `200`
- Expected: `267.50`
- Actual: `267.50`
- Result: ผ่าน

**ชุดที่ 3 — ยอดเยอะ โดนตัดเหลือ 1000**
10% ของ 20000 คือ 2000 ซึ่งเกินเพดานไปเท่าตัว
- Input: `20000`
- Expected: `22470.00`
- Actual: `22470.00`
- Result: ผ่าน

**ชุดที่ 4 — จุดตัดพอดีเป๊ะ**
500 บาท คิด 10% ได้ 50 พอดี ผมอยากรู้ว่าเงื่อนไข `<` ที่เขียนไว้จะไปดันค่าผิดไหม ปรากฏว่าไม่ เพราะ 50 ไม่ได้น้อยกว่า 50
- Input: `500`
- Expected: `588.50`
- Actual: `588.50`
- Result: ผ่าน

**ชุดที่ 5 — จุดตัดอีกฝั่ง**
10000 บาท คิด 10% ได้ 1000 พอดี ชนเพดานแบบไม่เกิน
- Input: `10000`
- Expected: `11770.00`
- Actual: `11770.00`
- Result: ผ่าน
