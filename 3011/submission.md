## STEP 1 อธิบายโจทย์

**โจทย์**
รับชื่อสีมา 2 บรรทัด แล้วบอกว่าถ้าเอาสองสีนั้นมาผสมกันจะได้สีอะไร โดยแม่สีมีแค่ Red Yellow Blue ถ้ามีสีไหนที่ไม่ใช่แม่สีโผล่มา ให้ตอบ Error

คู่สีที่โจทย์ให้มา
- แดง + เหลือง = Orange
- แดง + น้ำเงิน = Violet
- เหลือง + น้ำเงิน = Green

**Input**
2 บรรทัด บรรทัดละ 1 ชื่อสี

**Output**
1 บรรทัด เป็นชื่อสีที่ผสมได้ หรือคำว่า Error

**Constraints**
- Timeout 1 วินาที
- Memory 32000 bytes

## STEP 2 สิ่งที่สังเกต

อย่างแรกที่เห็นเลยคือคู่สีมันมีแค่ 3 คู่ ไม่ได้เยอะ เลยไม่ต้องคิดวิธีอะไรซับซ้อน เขียนเช็กตรง ๆ ก็จบ

แต่จุดที่กวนใจคือเรื่องลำดับ โจทย์บอกว่าแดง+เหลือง ได้ส้ม แต่ถ้าคนกรอกเหลืองก่อนแล้วค่อยแดง มันก็ต้องได้ส้มเหมือนกัน เพราะการผสมสีมันไม่สนว่าใส่อะไรก่อน เลยต้องเช็กทั้งสองทาง

อีกอย่างคือเรื่องตัวพิมพ์ โจทย์ไม่ได้บอกชัดว่า input จะมาเป็น red หรือ RED หรือ Red ผมเลยแปลงเป็นตัวใหญ่ทั้งหมดก่อนเทียบ จะได้ไม่ต้องมานั่งเดา ส่วนตอนพิมพ์ออกผมยึดตามที่โจทย์เขียน คือขึ้นต้นตัวใหญ่อย่างเดียว

edge case ที่นึกออก
- ใส่สีเดียวกันทั้งสองบรรทัด เช่น red กับ red
- ใส่สีที่ไม่ใช่แม่สี เช่น green หรือ pink
- ใส่ผิดทั้งสองบรรทัด
- สลับลำดับคู่สี

เคสสีเดียวกันนี่ผมยังไม่แน่ใจตรงนี้ว่าโจทย์ต้องการอะไร เพราะมันไม่ได้เขียนไว้ ผมเดาเอาว่าแดงผสมแดงก็ยังเป็นแดง เลยให้พิมพ์ชื่อสีนั้นออกไปเลย

## STEP 4 วิธีที่ใช้จริง

ทำตามแผนเดิมเกือบหมด ไม่ได้เปลี่ยนแนวคิด แต่มีปรับรายละเอียดสองจุด

จุดแรก ตอนแรกผมกะจะเขียน if ไล่เช็ก 6 แบบ คือ แดงเหลือง เหลืองแดง แดงน้ำเงิน น้ำเงินแดง เหลืองน้ำเงิน น้ำเงินเหลือง แต่เขียนไปสามอันแล้วรู้สึกว่ามันยาวเกินไป เลยรวมสองทิศทางไว้ใน if เดียวโดยใช้ or คั่น

จุดที่สอง ผมตัดเงื่อนไขคู่สุดท้ายทิ้ง แล้วใช้ else แทน เพราะพอเช็ก Error ไปแล้ว เช็กสีซ้ำไปแล้ว เช็กส้มกับเขียวไปแล้ว มันเหลือความเป็นไปได้แค่แบบเดียวคือแดงกับน้ำเงิน เลยไม่ต้องเขียนเงื่อนไขซ้ำอีก

การเอาเงื่อนไข Error ไว้บนสุดผมว่าช่วยได้เยอะ เพราะข้างล่างไม่ต้องกังวลเรื่องค่าแปลก ๆ อีกแล้ว

โค้ดที่ส่งไป

```python
mom = ["RED", "YELLOW", "BLUE"]

cone = input().upper()
ctwo = input().upper()

if cone not in mom or ctwo not in mom:
    print("Error")
else:
    if cone == ctwo:
        print(cone[0].upper()+cone[1:].lower())
    elif cone == mom[0] and ctwo == mom[1] or cone == mom[1] and ctwo == mom[0]:
        print("Orange")
    elif cone == mom[1] and ctwo == mom[2] or cone == mom[2] and ctwo == mom[1]:
        print("Green")
    else:
        print("Violet")
```

ตรง `cone[0].upper()+cone[1:].lower()` คือตอนเจอสีซ้ำ ผมเก็บ input ไว้เป็นตัวใหญ่หมดแล้ว ถ้าพิมพ์ตรง ๆ จะได้ RED ซึ่งหน้าตาไม่เหมือนสีอื่นที่พิมพ์ออกไป เลยต้องประกอบใหม่ให้เป็น Red

## STEP 5 Test Case

**ชุดที่ 1 — คู่ตรงตามโจทย์**
ลองอันที่ง่ายที่สุดก่อน ตรงตามตัวอย่างที่โจทย์เขียนไว้เป๊ะ ๆ
- Input: `Red` / `Yellow`
- Expected: `Orange`
- Actual: `Orange`
- Result: ผ่าน

**ชุดที่ 2 — สลับลำดับ**
เคสนี้สำคัญ เพราะถ้าเขียน if ไม่ครบสองทิศทาง โปรแกรมจะหลุดไปเข้า else แล้วตอบ Violet ทั้งที่ควรได้ Green ผมว่าคนน่าจะพลาดตรงนี้กันเยอะ
- Input: `blue` / `yellow`
- Expected: `Green`
- Actual: `Green`
- Result: ผ่าน

**ชุดที่ 3 — มีสีที่ไม่ใช่แม่สี**
เอา Green ซึ่งเป็นสีผสม ไม่ใช่แม่สี มาเป็น input ดู
- Input: `Green` / `Red`
- Expected: `Error`
- Actual: `Error`
- Result: ผ่าน

**ชุดที่ 4 — สีเดียวกัน**
เคสที่โจทย์ไม่ได้พูดถึง ผมลองเผื่อไว้

- Expected: `Blue`
- Actual: `Blue`
- Result: ผ่าน

**ชุดที่ 5 — พิมพ์ตัวเล็กตัวใหญ่ปนกัน**
อยากรู้ว่า `.upper()` ที่ใส่ไว้ทำงานจริงไหม

- Actual: `Violet`
- Result: ผ่าน
